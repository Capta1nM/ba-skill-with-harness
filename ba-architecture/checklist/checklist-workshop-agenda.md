# Checklist — Workshop Agenda

## Metadata required

- File path: `docs/elicitation/guides/workshop-<topic-slug>.md`
- Status: `draft | review | agreed`

## DoR (vào `review`)

- [ ] Topic + date + duration + facilitator + note-taker filled.
- [ ] Participants list with role + required/optional flag.
- [ ] Objectives ≥2.
- [ ] Expected outputs cụ thể (agreed process v0, prioritized list...).
- [ ] Agenda với time-box mỗi activity (tổng ≤ 2.5h).
- [ ] Mỗi activity có: format, owner, output.
- [ ] Materials needed + pre-read list.
- [ ] Ground rules: parking lot, async confirm decisions, one conversation.
- [ ] Post-workshop checklist.
- [ ] Prioritization activity (MoSCoW/dot vote) nếu bàn scope/feature.

## DoD (vào `agreed`)

- [ ] Facilitator dry-run agenda ít nhất 1 lần.
- [ ] Materials shared với participants ≥24h trước.

## Trace links bắt buộc

- Upstream: plan.md (workshop được plan), stakeholders.md (participants).
- Downstream: sessions/SES-*.md sau workshop, glossary candidates,
  raw statements parked sau.

## Anti-patterns

- ❌ Agenda quá tải (>2.5h liên tục, không break).
- ❌ Không có owner per-activity.
- ❌ Decisions trong workshop được coi là final (phải confirm văn bản
  sau).
- ❌ Skip parking lot → solution debates phá flow.
