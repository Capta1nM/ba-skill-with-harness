# Domain

Suggested layout:

```text
domain/
  glossary.md                    # project-specific business terms
  context-map.md                 # filled from templates/domain-context-map.md
  modules/
    MOD-<name>.md                # filled from templates/module-card.md
  scope.md                       # in-scope / out-of-scope with reasons
```

The project glossary extends `docs/GLOSSARY.md`; it does not replace harness
terminology.
