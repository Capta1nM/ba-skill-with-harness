# Harness Backlog

Use this file when an agent or BA discovers a missing harness capability but
should not change the operating model immediately.

## Template

```md
## Missing Harness Capability

### Title

Short name.

### Discovered While

Task or artifact that exposed the gap.

### Current Pain

What was hard, repeated, ambiguous, or unsafe?

### Suggested Improvement

What should be added or changed? (template, gate, rule, workflow phase,
matrix dimension, etc.)

### Risk

Tiny, normal, or high-risk.

### Status

proposed | accepted | implemented | rejected
```
