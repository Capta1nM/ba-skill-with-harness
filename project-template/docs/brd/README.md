# BRD

The Business Requirements Document for the loaded engagement lives here as
`brd.md`, drafted from `docs/templates/brd.md`.

If the engagement has multiple major releases or product lines, the BRD may
be split (e.g. `brd-core.md`, `brd-phase2.md`) with an `index.md` that names
the canonical pieces.

Sign-off evidence is recorded in `../validation/`, not inside the BRD body.
